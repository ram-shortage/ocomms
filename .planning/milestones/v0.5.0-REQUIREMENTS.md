# Requirements: OComms v0.5.0 Feature Completeness

**Defined:** 2026-01-20
**Core Value:** Data sovereignty - complete control over communication data

## v1 Requirements

Requirements for v0.5.0 release. Each maps to roadmap phases.

### User Status Messages

- [x] **STAT-01**: User can set custom status with emoji and text (up to 100 characters)
- [x] **STAT-02**: User status displays next to name in member lists and message headers
- [x] **STAT-03**: User can set status expiration time (auto-clear after duration)
- [x] **STAT-04**: User can select from preset status options (In a meeting, Out sick, On vacation, Focusing)
- [x] **STAT-05**: User can manually clear status immediately
- [x] **STAT-06**: User can pause notifications while status is active (DND mode)

### Bookmarks / Saved Messages

- [x] **BOOK-01**: User can save any message to personal saved list
- [x] **BOOK-02**: User can save files to personal saved list
- [x] **BOOK-03**: User can view all saved items in sidebar section
- [x] **BOOK-04**: User can remove item from saved list
- [x] **BOOK-05**: User can click saved item to jump to original context

### Scheduled Messages

- [x] **SCHD-01**: User can schedule message for specific date and time
- [x] **SCHD-02**: User can view list of scheduled messages
- [x] **SCHD-03**: User can edit scheduled message before send time
- [x] **SCHD-04**: User can cancel scheduled message
- [x] **SCHD-05**: Scheduled messages work in channels and DMs
- [x] **SCHD-06**: Scheduling is timezone-aware (displays in user's timezone)
- [x] **SCHD-07**: Quick-pick suggested send times available (Tomorrow 9am, Monday 9am)

### Reminders

- [x] **RMND-01**: User can set reminder on any message ("Remind me about this")
- [x] **RMND-02**: User can set reminder for specific date and time
- [x] **RMND-03**: User can view list of pending reminders
- [x] **RMND-04**: User can mark reminder as complete
- [x] **RMND-05**: User can snooze reminder when it fires
- [x] **RMND-06**: Reminder notification links to original message
- [x] **RMND-07**: User can create recurring reminders (daily, weekly)

### User Groups

- [x] **UGRP-01**: Admin can create named user group with handle (@designers)
- [x] **UGRP-02**: @mention of group notifies all group members in channel
- [x] **UGRP-03**: User can click group mention to view members
- [x] **UGRP-04**: Admin can add/remove members from group
- [x] **UGRP-05**: Group handles are unique within workspace
- [x] **UGRP-06**: Group mention only notifies members who are also in the channel

### Channel Categories

- [x] **CCAT-01**: Admin can create named channel categories
- [x] **CCAT-02**: Channels can be assigned to a category
- [x] **CCAT-03**: User can collapse/expand categories in sidebar
- [x] **CCAT-04**: User can drag channels between categories *(partial - UX issues tracked)*
- [x] **CCAT-05**: Uncategorized channels display in separate section
- [x] **CCAT-06**: Category sort order is configurable

### Link Previews / Unfurling

- [x] **LINK-01**: URLs in messages display Open Graph preview (title, description, image)
- [x] **LINK-02**: Preview shows for first 5 links in a message
- [x] **LINK-03**: Clicking preview opens URL in new tab
- [x] **LINK-04**: Previews are cached for performance
- [x] **LINK-05**: Twitter Card metadata used as fallback when Open Graph missing
- [x] **LINK-06**: User can dismiss/remove preview from message
- [x] **LINK-07**: Internal network URLs are blocked from fetching (SSRF protection)

### Typing Indicators

- [x] **TYPE-01**: Channel shows "[Name] is typing..." when user composes message
- [x] **TYPE-02**: Multiple simultaneous typers displayed (e.g., "Alice, Bob are typing...")
- [x] **TYPE-03**: Typing indicator auto-hides after 5 seconds of inactivity
- [x] **TYPE-04**: Typing indicator clears immediately when message is sent
- [x] **TYPE-05**: Typing events throttled to prevent broadcast storms (max 1 per 3 seconds)

### Custom Emoji

- [x] **EMOJ-01**: User can upload custom emoji image (PNG, JPG, GIF up to 128KB)
- [x] **EMOJ-02**: Custom emoji usable in messages with :name: syntax
- [x] **EMOJ-03**: Custom emoji usable in reactions
- [x] **EMOJ-04**: Emoji names are unique within workspace
- [x] **EMOJ-05**: Custom emoji visible in dedicated tab in emoji picker
- [x] **EMOJ-06**: User can delete own uploaded emoji
- [x] **EMOJ-07**: Animated GIF emoji supported
- [x] **EMOJ-08**: SVG uploads converted to PNG for XSS protection

### Channel Archiving

- [x] **ARCH-01**: Admin/creator can archive channel (sets read-only)
- [x] **ARCH-02**: Archived channels remain searchable
- [x] **ARCH-03**: Archived channels hidden from main sidebar
- [x] **ARCH-04**: User can browse list of archived channels
- [x] **ARCH-05**: Admin can unarchive channel (restores normal operation)
- [x] **ARCH-06**: Default channel (#general) cannot be archived

### Guest Accounts

- [x] **GUST-01**: Admin can invite external user as guest
- [x] **GUST-02**: Guest access restricted to specified channels only
- [x] **GUST-03**: Guest badge displays on profile and in member lists
- [x] **GUST-04**: Guest can send messages, react, upload files in allowed channels
- [x] **GUST-05**: Admin can remove guest access
- [x] **GUST-06**: Guest cannot see workspace member directory (only channel members)
- [x] **GUST-07**: Guest cannot be added to user groups
- [x] **GUST-08**: Admin can set guest expiration date (auto-deactivate)

### Workspace Analytics

- [x] **ANLY-01**: Admin dashboard shows message volume over time
- [x] **ANLY-02**: Dashboard shows active users (DAU/WAU/MAU)
- [x] **ANLY-03**: Dashboard shows channel activity ranking
- [x] **ANLY-04**: Admin can filter analytics by date range
- [x] **ANLY-05**: Admin can export analytics to CSV
- [x] **ANLY-06**: Dashboard shows peak usage times (hourly distribution)
- [x] **ANLY-07**: Dashboard shows file storage usage
- [x] **ANLY-08**: Analytics data is aggregate only (no individual user surveillance)

### Testing & Stabilization

- [x] **TEST-01**: Unit tests for all new v0.5.0 backend services
- [x] **TEST-02**: Integration tests for cross-feature interactions
- [x] **TEST-03**: Socket.IO event tests for real-time features
- [x] **TEST-04**: API endpoint tests for all new routes
- [x] **TEST-05**: Authorization tests for guest account restrictions
- [ ] **TEST-06**: Comprehensive test coverage for all existing features (v0.1-v0.4) *(descoped)*
- [x] **TEST-07**: Bug fixes for issues discovered during testing
- [ ] **TEST-08**: Performance testing for typing indicators at scale *(descoped)*

## Out of Scope

Explicitly excluded with reasoning.

| Feature | Reason |
|---------|--------|
| Calendar integration for status | External service dependency, violates self-hosted principle |
| Rich presence (app/music sharing) | Privacy concerns, not relevant for work context |
| Read receipts | Privacy nightmare, user backlash |
| "Last seen" timestamps | Surveillance feeling |
| Nested channel categories | Complexity trap, Discord uses single level |
| Per-category permissions | Significant auth complexity, defer to future |
| Natural language reminder parsing | High error rate, use explicit date picker |
| Auto-play video in link previews | Bandwidth/annoyance |
| Scheduled thread replies | Odd UX, Slack doesn't support |
| Recurring scheduled messages | Workflow builder territory |
| Per-user activity leaderboards | Gamification backfire |
| Guests in user groups | Slack explicitly prevents this |
